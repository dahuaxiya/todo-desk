# Todo Desk：把桌面待办、AI Agent 工作和飞书协同放到一个地方

Todo Desk 是一个便笺风格的桌面 Todo 工具，设计目标不是再做一个复杂的项目管理系统，而是让用户在写代码、沟通、排期和让 AI agent 处理工作时，能快速记录“我现在正在做什么、还有什么要做、哪些已经完成”。它支持本地存储、飞书同步、日历提醒、图片附件和本机 API，也能让 Codex、Claude、Kimi、Cursor 等 agent 自动把工作进展挂到同一个任务面板里。

它更适合放在桌面边缘常驻使用：平时像便笺一样轻量停靠，需要时展开看详情；任务变多时可以搜索、按来源分类、按项目或优先级梳理；工作完成后可以同步到飞书文档和日历，减少手工整理。

## 它解决什么问题

很多工作不是完整项目，而是散落在几个地方的小任务：一条飞书消息、一次代码修改、一个 AI agent 正在跑的 session、一个截图里识别出来的待办，或者一个需要在晚上提醒的事项。传统 Todo 工具能记录任务，但很少理解“AI 正在做的工作”和“本地桌面上下文”；项目管理工具又太重，不适合一直贴在桌面上。

Todo Desk 关注三个问题：

- 当前工作要足够容易记录，最好不用切换太多窗口。
- 任务要能和 AI agent、代码库、session 绑定，后续能追溯谁创建、处理到了哪一步。
- 本地桌面、飞书文档、飞书日历和系统日历要尽量保持一致，任务完成或删除后不需要重复维护。

## 核心使用方式

Todo Desk 的主界面分为三类任务：正在做、Todo、已完成。用户可以直接添加任务，也可以使用 AI 添加，把自然语言、截图或粘贴板图片交给 AI 解析成结构化任务。

每个任务可以记录标题、详情、状态、优先级、项目/分组、标签、截止时间、提醒时间和图片附件。任务卡片支持展开详情、复制内容、编辑、完成、删除和恢复。删除后的任务会进入回收站，不会立刻丢失。

对日常使用来说，典型流程是：

1. 把正在处理的事项添加到“正在做”。
2. 给需要提醒的事项设置截止时间或提醒时间。
3. 工作推进时编辑详情、补充图片或标签。
4. 完成后勾选任务，必要时同步到飞书文档。
5. 删除任务时，同步清理本地日历和飞书日历里的对应事件。

## 桌面形态

Todo Desk 支持正常窗口、小卡模式、贴附模式和置顶。正常窗口适合集中整理任务；小卡模式适合只看一个列表；贴附模式适合长期贴在屏幕左右边缘，减少遮挡其他窗口。

当前版本支持全局快捷键，并且可以在设置中自定义：

| 动作 | 默认快捷键 |
| --- | --- |
| 切换贴附 / 恢复 | ⌘⇧T |
| 贴附到左侧 | ⌘⇧← |
| 贴附到右侧 | ⌘⇧→ |
| 小卡模式 | ⌘⇧M |
| 置顶 | ⌘⇧P |

设置快捷键时，应用会临时暂停当前全局快捷键，避免录入过程中误触发窗口动作。也可以一键恢复默认快捷键。

## AI 能力

Todo Desk 内置 AI 解析流程，适合把不规整的输入转换成任务：

- 从自然语言里识别标题、详情、状态、优先级、项目、标签、截止时间和提醒时间。
- 添加任务时可以粘贴图片，优先使用模型视觉能力；如果模型不支持图片，则退回 OCR，再把识别文字交给同一套解析流程。
- 编辑任务时可以输入修改要求，让 AI 在当前任务草稿上做调整，例如“改成高优先级，明天 10 点提醒，详情整理成步骤”。
- 多选任务后可以合并，支持普通拼接，也支持 AI 合并成更清晰的一条任务。

AI 能力使用兼容 OpenAI Chat Completions 的接口，在设置里配置 Base URL、Model 和 API Key 即可。没有开启 AI 时，普通添加任务仍然可用。

## 飞书和日历同步

Todo Desk 通过本机 `lark-cli` 接入飞书。用户在设置里填入飞书文档 URL 或 token 后，完成任务时可以把任务快照追加到飞书文档。这样适合做日报、工作记录和 agent 工作留痕。

有时间的任务还可以同步到日历：

- 自动加入 macOS 系统日历。
- 自动同步到飞书日历。
- Todo Desk 内部也提供日历视图，方便按日期查看任务。

删除任务时会尽量执行强同步：如果任务已经创建过本地日历事件或飞书日历事件，删除任务时会同步删除对应事件。恢复回收站任务时会清理旧同步状态，再按当前时间重新创建日历事件。

## Agent 自动接入

Todo Desk 提供本机 API 和 agent skill，让 AI coding agent 可以把自己的工作写入 Todo Desk。应用启动后默认监听：

```text
http://127.0.0.1:47731
```

agent 创建任务时会带上来源信息，例如：

- agent 名称：codex、claude、kimi、cursor 等。
- 当前 session id。
- 当前代码库名称和路径。
- 任务来源 `origin.kind = agent`。

这些字段不是装饰信息，而是后续筛选、追溯和样式区分的依据。人工创建的任务和 agent 创建的任务会被清楚地区分，避免误判。

仓库内提供自动安装脚本：

```bash
npm run agent:install -- --dry-run
npm run agent:install
```

它会把 Todo Desk skill 和工作挂载规则安装到 Codex、Claude、Kimi、Cursor 等 agent 的全局配置里。安装后，agent 开始处理明确任务时会先创建 Todo Desk 任务；推进时更新同一条任务；实现完成但用户还没确认时，只能请求完成审批，不能自己把任务标记为 done。

## 本地数据和迁移

Todo Desk 默认使用本地 JSON 存储，方便备份和迁移。macOS 下默认目录是：

```text
~/Library/Application Support/todo-desk/
```

主要文件包括：

- `todo-desk-data.json`：任务、回收站、同步记录、设置和窗口状态。
- `attachments/`：任务图片附件。
- `calendar-events/`：本地日历同步相关文件。
- `todo-desk.log`：运行日志和同步/快捷键注册等诊断信息。

这种存储方式的好处是透明、可备份、可迁移。未来如果需要多端同步，也可以在这个基础上继续扩展云端存储。

## 适合谁使用

Todo Desk 适合经常在桌面端处理多条并行事项的人，尤其适合以下场景：

- 开发者：同时处理代码、排查、发布、Review 和临时需求。
- AI agent 重度使用者：希望记录每个 agent session 正在做什么、做到哪一步。
- 需要飞书留痕的人：希望完成事项自动同步到飞书文档和日历。
- 习惯便笺式工具的人：希望待办常驻桌面边缘，而不是藏在一个完整项目管理系统里。

## 当前边界

Todo Desk 是桌面优先的工具。Web 降级模式可以展示界面，但不包含完整的原生能力，例如系统通知、本地 OCR、文件存储、窗口贴附和全局快捷键。飞书同步依赖本机 `lark-cli` 的登录和授权状态；如果飞书权限不足，对应同步会失败并写入日志。

跨端方向已经预留，但当前主要体验仍然集中在 macOS 桌面端。后续可以继续补充移动端查看、云端数据同步、MCP 工具层和更完整的团队协作能力。

## 一句话

Todo Desk 的核心价值是：把“我在做什么”“AI 在做什么”“什么时候提醒”“完成后同步到哪里”放到一个轻量桌面面板里，让个人工作流和 AI 工作流有一个共同的记录入口。
